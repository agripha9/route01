## 6. 로딩 애니메이션 사양 (2026-04-21 확정)

- **구조**: 일직선, viewBox `0 0 260 20`, 높이 20px (한 줄 높이)
- **양 끝**: `0` (x=6, 네이비) / `1` (x=254, 크림슨), font-weight 800, 15px
- **중간**: 점 15개(r=1.8, `#3a5a9e`) + 노드 4개(r=3, 그라디언트 4색)
- **순서**: `0` → 점·점·점 → 노드1 → 점·점·점 → 노드2 → 점·점·점 → 노드3 → 점·점·점 → 노드4 → 점·점·점 → `1`
- **등장**: 22단계 누적 fade-in (`rl-s0` ~ `rl-s21`, 각 120ms 간격)
- **총 길이**: 2.64초, 끝나면 모두 fade out 후 0부터 재시작
- **CSS 클래스**: `.route-loader`, `.rl-end`, `.rl-dot`, `.rl-node`, `.rl-step`, `.rl-s0..21`
- **키프레임**: `@keyframes rl-step` (0% → 2% pop → 5~92% hold → 96% fade out)

## 8. 알려진 이슈 (우선순위 순)

1. **답변 화면 내 네이비/파란 선이 간간이 보임** — blockquote는 `#d2d2d7` 회색으로 설정돼 있는데 실제 렌더에서 네이비처럼 보인다는 피드백. h3/h4/블록 요소 색 leakage 여부 미확인. 브라우저 개발자도구로 태그·color 값 확인 필요.
2. **숫자 리스트 들여쓰기 불일치** — `1.` `2.` 순번 정렬이 안 맞는 경우 있음.
3. **답변·docx·pdf 표 높이 차이** — 세 렌더 경로의 셀 padding·line-height 일치 필요.
